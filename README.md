# Прогнозирование рецидивов диабетического кетоацидоза

Научно-исследовательская работа: прогноз повторного эпизода диабетического
кетоацидоза (ДКА) по клиническим данным пациента. Итог - препринт статьи и
прототип онлайн-сервиса поддержки решений врача.

Когорта: 289 пациентов одного центра, из них 273 с известным исходом (рецидив или
единичный эпизод). Доля рецидивов около 32%. Задача - бинарная классификация при
малой несбалансированной выборке, акцент на честную валидацию и клиническую
применимость, а не на завышенные метрики.

## Краткие результаты

- Разделяющая способность: по честной вложенной кросс-валидации 0.72-0.78 (лидер
  случайный лес на наборе без мультиколлинеарности, 0.775), на отложенном тесте все
  финалисты в коридоре 0.72-0.80 с широкими (около ±0.13) перекрывающимися
  доверительными интервалами. Из-за малой выборки модели разных семейств
  статистически неразличимы.
- Подобраны восемь моделей-финалистов: четыре семейства (логистическая регрессия,
  случайный лес, XGBoost, CatBoost) на двух наборах признаков. На тесте они
  неразличимы, поэтому единую модель не выделяем, а в сервис выносим все восемь с
  выбором за врачом. Логистическая регрессия при этом ценна как самая простая,
  интерпретируемая и хорошо калиброванная.
- Синтетическая балансировка (SMOTE и варианты), стекинг и сложные модели прироста
  ROC-AUC не дали. Балансировка и порог управляют разменом чувствительности и
  специфичности, а не разделяющей способностью.
- Порог подбирали под целевую чувствительность не ниже 0.75 для каждого финалиста: в
  этом режиме модели работают как скрининг, с высоким NPV (0.75-0.92) при умеренном
  PPV.
- Значимые предикторы согласуются между классической статистикой (Таблица 1,
  поправка Бенджамини-Хохберга) и SHAP: суточная доза инсулина, вид инсулинотерапии,
  длительность СД, HbA1c, ЛПВП, ХБП (стадия и альбуминурия), ретинопатия,
  диабетическая нейропатия, прием алкоголя.
- Проверены две заранее заданные гипотезы. Клиническая: прием алкоголя за сутки до
  эпизода связан с рецидивом - подтверждена (ОШ 2.27, p=0.008). Методологическая:
  сложные модели превосходят логистическую регрессию по ROC-AUC - отклонена (даже
  лучшая модель против логрега дает DeLong p=0.05, и преимущество не держится на тесте).

Числа приведены как ориентир; полное описание с интервалами - в препринте
(`paper/preprint.md`).

## Данные и доступ

Репозиторий открытый, данные пациентов закрыты. Сырые данные
(`data/raw/dka_bd_289.xlsx`), промежуточные и обработанные таблицы, обученные
модели и диагностические отчеты с идентификаторами не публикуются (см.
`.gitignore`). Это соответствует требованиям защиты персональных медицинских
данных.

Воспроизведение полного пайплайна требует исходного датасета, размещенного по
пути `data/raw/dka_bd_289.xlsx`. Доступ к данным - по обоснованному запросу к
авторам и держателю данных, в рамках этических ограничений.

## Методология

- Профилактика утечки данных: разбиение train/test делается один раз и рано;
  импутация, масштабирование, кодирование, отбор и синтетика обучаются только на
  train внутри фолдов кросс-валидации; отложенный тест трогается один раз.
- Честная валидация при малой выборке: стратифицированная кросс-валидация,
  вложенная кросс-валидация при подборе гиперпараметров, доверительные интервалы
  метрик (бутстрэп).
- Метрики под клинику и дисбаланс: ROC-AUC, PR-AUC, чувствительность,
  специфичность, калибровка (Brier, калибровочные кривые, Платт и изотоническая),
  анализ кривых принятия решений (decision curve analysis), оптимизация порога.
- Статистика по фиксированному протоколу медицинского статистика (критерии
  нормальности, выбор тестов, отношение шансов с 95% ДИ, поправка на множественные
  сравнения). Протокол - в `docs/methodology.md`.
- Отчетность ориентирована на TRIPOD-AI (стандарт описания клинических
  прогностических моделей на основе ИИ).

## Структура репозитория

- `src/` - переиспользуемый код:
  - `config.py` - пути, seed, константы;
  - `io.py`, `cleaning.py`, `columns.py` - загрузка, очистка, классификация колонок;
  - `stats.py`, `tables.py` - статистический протокол и Таблица 1;
  - `eda.py`, `eda_overview.py`, `viz.py` - разведочный анализ и графики;
  - `missingness.py`, `imputation.py`, `manual_impute.py` - анализ и заполнение
    пропусков (автоматические стратегии и ручная клиническая импутация);
  - `dimreduce.py` - снижение размерности;
  - `features.py`, `modeling.py`, `tuning.py`, `optuna_tuning.py` - подготовка
    признаков, пайплайны, тюнинг с вложенной CV (Optuna - рабочий, для финалистов);
  - `evaluation.py`, `threshold.py`, `interpret.py`, `bootstrap_ci.py`,
    `shap_compare.py` - оценка на тесте, порог, SHAP, бутстрэп-интервалы, сверка SHAP
    со встроенной важностью;
  - `ensembles.py`, `synthetic.py` - стекинг и синтетическая балансировка;
  - `hypothesis_test.py` - проверка методологической гипотезы (DeLong, парный бутстрэп);
  - `service_model.py` - финальные модели для сервиса.
- `notebooks/` - тонкие презентационные слои над `src/` по этапам (01-15).
- `service/app.py` - онлайн-сервис на Streamlit.
- `reports/` - графики, таблицы, отчеты (материалы с данными пациентов не
  коммитятся).
- `models/` - обученные модели сервиса (не коммитятся, регенерируются из данных).
- `docs/methodology.md` - методология проекта: стек, статистический протокол,
  сквозные правила ML.
- `paper/` - научный препринт: текст (`preprint.md`), исходники LaTeX (`overleaf/`),
  чек-лист отчетности TRIPOD-AI.

## Окружение и воспроизводимость

Менеджер окружений - conda, окружение `dka` (Python 3.11). Версии пакетов
зафиксированы; источник истины - `requirements.txt` (все ставится через pip),
`environment.yml` создает python и pip и ставит из requirements.

```
conda env create -f environment.yml
conda activate dka
```

Зерно случайности одно на весь проект (`RANDOM_SEED` в `src/config.py`),
используется в разбиении, кросс-валидации, моделях и синтетике.

## Как воспроизвести пайплайн

После размещения датасета в `data/raw/` модули запускаются как пакеты, например:

```
python -m src.cleaning        # очистка -> data/processed/dka_clean.parquet
python -m src.tables          # Таблица 1 по группам
python -m src.missingness     # анализ природы пропусков
python -m src.imputation      # стратегии импутации и сравнение распределений
python -m src.manual_impute   # ручная клиническая импутация и сравнение качества
python -m src.modeling        # сетка моделей (CV на train)
python -m src.optuna_tuning   # вложенная CV (Optuna), честная оценка финалистов
python -m src.evaluation      # оценка на отложенном тесте
python -m src.threshold       # оптимизация порога
python -m src.interpret       # SHAP по финалистам
python -m src.bootstrap_ci    # бутстрэп-интервалы и OOB
python -m src.ensembles       # стекинг финалистов
python -m src.synthetic       # синтетическая балансировка
python -m src.shap_compare    # сверка SHAP со встроенной важностью
python -m src.hypothesis_test # проверка гипотезы о сложности модели (DeLong)
python -m src.service_model   # обучение моделей сервиса -> models/
```

Ноутбуки в `notebooks/` дублируют этапы для просмотра; выводы очищаем перед
коммитом.

## Онлайн-сервис

Прототип поддержки решений врача на финальных моделях:

```
python -m streamlit run service/app.py
```

Демонстрация работы:

<video src="https://github.com/ovelsad/dka-relapse-ml-prediction/raw/master/docs/streamlit_demo.webm" controls width="100%"></video>

Если видео не воспроизводится в браузере, оно доступно файлом:
[docs/streamlit_demo.webm](docs/streamlit_demo.webm).

Сервис дает выбор модели (логистическая регрессия, случайный лес, XGBoost, CatBoost)
и набора признаков (значимые из 10, без мультиколлинеарности из 25), показывает
калиброванную вероятность рецидива, относит пациента к группе риска по
регулируемому порогу с пояснением чувствительности и специфичности, поясняет вклад
показателей (коэффициенты для логрега, SHAP для деревьев) и приводит карточку
модели с ограничениями. Незаполненные поля заполняет импутер пайплайна, об этом
сервис предупреждает. Все вычисления локальные, данные наружу не передаются.

## Ограничения

Одноцентровая выборка небольшого размера. Метрики имеют широкие доверительные
интервалы. Модель не валидирована на внешней независимой когорте. Часть
лабораторных показателей имеет высокую долю пропусков. Сервис - вспомогательный
инструмент и прототип, он не заменяет клиническое решение и требует внешней
валидации перед применением.
